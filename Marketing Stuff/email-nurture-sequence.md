# Email Nurture Sequence — JMS Dev Lab

## Platform Recommendation

**MailerLite** (https://www.mailerlite.com/) — Free up to 1,000 subscribers.

- Simple automation builder for drip sequences
- Good deliverability
- RSS-to-email feature for monthly blog digests
- No credit card required for free tier
- GDPR compliance built in (important for EU subscribers)

**Alternative: Buttondown** (https://buttondown.com/) — Free up to 100 subscribers.

- Developer-friendly, markdown-based
- Extremely simple interface
- Good if you prefer writing emails in plain text / markdown
- Limited automation on free tier (may not support full drip sequence without manual scheduling)

**Recommendation:** Start with MailerLite. It handles the full 5-email automation on the free tier and scales to 1,000 subscribers before you need to pay anything.

## Monthly Digest Setup

After the 5-email sequence completes, move subscribers to a monthly RSS-to-email digest.

### MailerLite setup:
1. Go to **Automations** > **Create automation**
2. After the 5-email sequence ends, add a step to tag subscribers with "nurture-complete"
3. Go to **Campaigns** > **RSS campaigns**
4. Set RSS feed URL: use the blog page URL or create a simple RSS feed at jmsdevlab.com/feed.xml
5. Set frequency: **Monthly** (first week of each month)
6. Template: Simple, text-focused layout matching the tone of the nurture emails
7. Include: Post title, first 2-3 sentences, link to read more
8. Subject line format: "New from JMS Dev Lab: [latest post title]"
9. From: JMS Dev Lab / hello@jmsdevlab.com

### If no RSS feed exists:
Create a simple manual monthly email when new posts are published. Keep it brief:
- 1-2 sentence intro
- Links to new blog posts with a one-line description each
- No fluff, no padding if there's nothing new

### Tone guidelines for all emails:
- Write like you're emailing a colleague, not a prospect
- No fake urgency ("limited time!", "act now!")
- No hype language ("revolutionary", "game-changing")
- Be honest about what you do and don't do
- If you recommend against buying, say so
- Short paragraphs, plain language, no jargon
- Every email should be useful even if the reader never buys anything

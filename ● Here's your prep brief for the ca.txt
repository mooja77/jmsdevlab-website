● Prototype — deeper detail

  The 4-tier classification pipeline:

  1. Direct lookup — exact match against 6,413 ingredients in a local database. Handles ~85% of ingredients instantly.
  Each entry has aliases (e.g. "E120" = "carmine" = "cochineal"), so it catches different label formats.
  2. Fuzzy matching — handles OCR errors and typos. "Gelatln" matches to "gelatin", "leclthin" matches to "lecithin".
  Uses Levenshtein distance scoring so near-misses don't slip through.
  3. Pattern recognition — 27+ heuristic rules that break down compound ingredients. "Hydrolysed vegetable protein" →
  recognises "vegetable" → vegan. "Milk chocolate powder" → recognises "milk" → not vegan. Covers dairy, eggs, meat,
  honey, E-numbers, plant-based qualifiers, etc.
  4. Cloud AI fallback — for the rare ingredient none of the above can classify (~2-3%), it calls an LLM API for
  analysis. The result gets cached locally so it's instant next time. Only triggers when online.

  Steps 1-3 run entirely on-device — no internet needed. This is critical for supermarket use where signal is
  unreliable.

  OCR / Camera — what the full MVP will deliver

  Camera capture:
  - React Native Camera module for live viewfinder
  - User points phone at ingredient label, taps to capture
  - Handles real-world conditions: curved packaging, small text, poor supermarket lighting, reflective surfaces

  Text extraction:
  - Google ML Kit (preferred) — on-device text recognition, no network needed, fast, free, works on iOS and Android.
  Better accuracy than Tesseract for structured text on packaging
  - Tesseract as fallback option — open source, fully offline, but lower accuracy on curved/distorted labels
  - Post-processing: strip line breaks, normalise separators (commas, semicolons, "contains:"), handle multi-language
  labels

  The pipeline after OCR:
  Camera → ML Kit extracts text → parser splits into individual ingredients → each ingredient goes through the 4-tier
  classification → results displayed with traffic-light UI (green/amber/red) and explainable reasoning for every
  ingredient

  Why ML Kit over Tesseract:
  - ML Kit is optimised for mobile — uses device GPU/NPU for acceleration
  - Higher accuracy on real-world photos (trained on millions of images)
  - Handles skewed/rotated text better
  - Zero latency — runs on-device
  - Tesseract is a desktop-era engine ported to mobile — works but slower and less accurate on packaging

  Edge cases the prototype already handles that carry into OCR:
  - "May contain traces of..." — flagged separately as cross-contamination risk, not classified as an ingredient
  - Nested brackets: "Chocolate (cocoa mass, sugar, cocoa butter, soy lecithin)"
  - E-numbers with and without the "E" prefix
  - Regional name variations (coriander/cilantro, aubergine/eggplant)
